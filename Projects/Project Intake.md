# Project Intake

Inbox for **possible** work before it earns a hub note on [[Projects/Projects Dashboard]]. If it is not a finite outcome with a clear “done,” it probably belongs in [[Areas/README]] or [[Resources/README]] instead.

## When to promote

Promote when you can answer **all** of:

1. **Outcome** — What changes in the world when this is finished?
2. **Owner** — Who drives it (even if that is you)?
3. **Next physical step** — One concrete action, not “think about it.”

If (2) or (3) is missing, keep it here or park it below.

## Triage queue

Duplicate the block per idea; check items off as you clarify.

- [ ] **Idea / label:**
  - **Desired outcome:** (one sentence)
  - **Non-goals:** (what we are not doing)
  - **Deadline / time box:** (if any)
  - **Priority:** (p0–p3 or low/med/high)
  - **Stakeholders / dependencies:**
  - **Next step:** (who does what by when)
  - **Decision:** promote · park · kill

## Parked (no active next step)

Short list of “maybe later” — no frontmatter required.

- _Add bullets when deferring without a hub note._

## Promote to project checklist

- [ ] Create a hub note from [[_System/Templates/Project Template]] in `Projects/` (same folder as this note’s parent — the vault `Projects/` root).
- [ ] Fill **Outcome** and at least one **Next Actions** checkbox.
- [ ] Set frontmatter `status` to `active`, `waiting`, or `on-hold` (`done` when finished).
- [ ] Set `due:` if there is a real deadline (optional but improves [[Projects/Projects Dashboard]] sorting).
- [ ] When closing out: set `status: done` and `completed: YYYY-MM-DD`.
- [ ] Prefer **one** entry point per project: **either** `Projects/<Name>.md` **or** `Projects/<Name>/MOC.md` (see [[_System/Templates/Project MOC Template]]), not both namesakes. Folder projects: put the hub in `MOC.md`; [[Projects/Projects Dashboard]] lists those automatically.

## After promote

- [ ] Link related Areas/Resources notes from the new hub.
- [ ] Remove or strikethrough the idea’s triage block here so the inbox stays honest.
